## 機能一覧(使い方ガイド)

### 画面構成

- **左レール** … ツール切替
- **上段バー** … 戻す/進む・書き出し・全消去、塗り/線/ブラシなどのスタイル設定
- **右パネル** … レイヤー・オブジェクト操作・パスファインダー・整列・変形・パス・シンボル・アートボード(見出しタップで開閉)

### ツール(左レール)

| ツール | 使い方 |
|---|---|
| 選択 | タップで選択、ドラッグで移動(他オブジェクトの端・中心にスナップ)。空白からドラッグで複数選択(マーキー)。角ハンドル=拡大縮小、上の丸ハンドル=回転 |
| ダイレクト | パスをタップ→アンカー/ハンドルをドラッグして形を編集 |
| ブラシ | 筆圧対応フリーハンド。種類(標準/カリグラフィ/マーカー/鉛筆/パターン点描)と太さは上段バーで変更 |
| 消しゴム | なぞった範囲を消す。塗り/閉じたパスは部分消し(型抜き)、開いた線・テキスト・グループは触れると丸ごと削除。太さはブラシサイズ連動(2倍) |
| ペン | タップでアンカー追加、ドラッグでハンドル(曲線)。始点をタップで閉じて確定。ツール切替でも確定 |
| テキスト | 空白タップで入力、既存テキストをタップで編集(空にすると削除)。サイズは上段バー「文字」 |
| 長方形/楕円/直線 | 対角にドラッグ |
| 多角形/星 | 中心から外へドラッグ(ドラッグ方向に回転)。多角形=6角・星=5芒 |
| スポイト | タップしたオブジェクトの塗り/線色を取得 |

### 指のジェスチャ(Apple Pencil と併用)

- **1本指ドラッグ** … キャンバス移動(パン)。指では描画されません(パームリジェクション)
- **ピンチ** … ズーム
- **2本指タップ** … 取り消し(Undo)

### スタイル(上段バー)

- 塗り: ON/OFF・色・種類(単色/線形グラデ/円形グラデ)・グラデ終点色
- 線: ON/OFF・色・線幅・線端(丸/なし/角)・角(丸/尖/面取)・破線
- 不透明度、描画モード(乗算・スクリーンなど7種)
- スウォッチ: タップで塗り色に適用、「＋」で現在色を登録(端末に保存)
- **選択中のオブジェクトがあれば、スタイル変更は即座にそれへ反映されます**

### 右パネル

- **レイヤー** … 追加/削除/表示切替/▲▼で並べ替え/タップで作業レイヤー切替
- **オブジェクト** … グループ化/解除、複製、削除、コピー/貼付、クリッピングマスク(最前面の形で切り抜き)、シンボル化
- **パスファインダー** … 合体/前面型抜き/交差/中マド(2つ以上選択して実行)
- **整列・分布** … 左右上下・中央揃え、横/縦等分(分布は3つ以上)
- **変形** … 左右/上下反転、シアー(段階適用)
- **パス** … スムーズ、簡略化、メッシュ風塗り(閉パスに多色グラデ)、画像トレース(写真→ベクター変換)
- **シンボル** … 登録済みシンボルをタップで画面中央に配置
- **アートボード** … 追加(800×600の白面)/タップでジャンプ/削除/その領域だけをSVG書出

### 保存と書き出し

- 編集は**自動保存**(端末内 IndexedDB)。アプリを閉じても次回復元されます
- 書き出し: SVG(ベクター)/ PNG(画面表示のまま)/ PDF(ベクター保持)

## 技術スタック

Vite + TypeScript / Paper.js(ベクターエンジン) / perfect-freehand(筆圧ストローク) / idb(IndexedDB) / Pointer Events API

## 開発

```bash
npm install

npm run build   # 型チェック + 本番ビルド → dist/
```

## iPad で動かす手順

### 開発中の確認(PC + iPad が同じ Wi-Fi)

1. PC で `npm run dev` を実行(`--host` 付きなので LAN に公開される)
2. 起動ログに表示される `Network: http://<PCのIP>:5173` の URL を控える
   (IP がわからない場合は Windows なら `ipconfig`、Mac なら `ifconfig` で確認)
3. iPad の Safari でその URL を開く
4. Apple Pencil で描いて動作確認

### 本番用(ホーム画面のアプリとして使う)

Service Worker(オフライン機能)は HTTPS が必須のため、無料の静的ホスティングにデプロイします。以下は Cloudflare Pages の例(Netlify / Vercel / GitHub Pages でも同様):

1. `npm run build` で `dist/` に本番ビルドを生成
2. Cloudflare Pages にデプロイ:
   - [dash.cloudflare.com](https://dash.cloudflare.com) → Workers & Pages → Create → Pages
   - 「Direct Upload」で `dist/` フォルダをアップロード(または GitHub 連携で自動デプロイ)
   - `https://<プロジェクト名>.pages.dev` が発行される
3. iPad の Safari でその URL を開く
4. **共有ボタン → 「ホーム画面に追加」** でアプリ化
5. 以後はホーム画面のアイコンから全画面で起動でき、一度読み込めばオフラインでも動作します

### (任意)ネイティブアプリ化

Capacitor でラップして Mac + Xcode から iPad にインストールする方法もあります。

## 構成

```
src/
├── main.ts            # 起動・各モジュールの配線
├── input/pointer.ts   # Pointer Events(ペン優先・ジェスチャ・coalesced events)
├── tools/brush.ts     # 筆圧ブラシ(perfect-freehand)
├── engine/history.ts  # Undo/Redo(スナップショット方式)
├── store/db.ts        # IndexedDB 永続化
└── ui/panels.ts       # ツールバー・レイヤーパネル
```

設計メモ: 独自のドキュメントモデルは持たず、Paper.js の project(Layer→Item)を
そのままドキュメントとして扱う。保存は `exportJSON`、書き出しは `exportSVG`。
